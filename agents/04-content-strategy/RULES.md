# RULES：产出合格门槛

> 本文件定义 Agent 04 的产出必须满足的硬性红线。任何输出如有任一条未满足，即为不合格，必须重做。
> RULES 只管"产出长什么样"，不管"怎么做出来"（工作流程见 DATAFLOW）。

---

## 一、季度策略必须满足

**R1. 分层完整**
输出必须同时包含三层：季度方向 → 月度重点 → 周度选题。缺任何一层都不合格。

**R2. 依据可追溯**
季度方向必须引用 intel-pool 或 insight-pool 中的具体条目作为依据，不得凭空给出。引用必须能在协作池中对应到原始条目。

**R3. 需求池响应**
策略必须说明"本季度需求池存量（来自 collaboration/demand-pool/approved/）是什么水平、质量如何（是否集中于某几个支柱/决策人）、因此爆款拆解/需求池自驱/外部热点的配比是多少"。不得忽略当前需求池状况。

**R4. 业务目标挂钩**
季度方向必须说明这个方向"打的是哪一类目标客户（T1/T2/T3），推动的是哪个产品（L1/L2/L3）"。不得是无业务指向的内容方向。

---

## 二、周度选题计划必须满足

**R5. 三要素齐全**
每一条选题必须包含且仅包含这三样：
- **来源**：需求库 / 爆款拆解 / 外部热点，三选一并标注
- **业务价值**：一句话说清"为什么做这条选题、它给业务带来什么"
- **钩子**：一句话说清"这条选题打动读者的点是什么"

缺任一要素，该条选题不合格。

**R6. 来源配比匹配阶段**
周选题单中三类来源（需求库/爆款拆解/外部热点）的配比，必须符合 STAGE_PLAYBOOK.md 中当前账号阶段对应的配比。偏离超过 ±20% 不合格。

**R7. 爆款拆解标注源头**
凡是"爆款拆解"来源的选题，必须标注：
- 参考爆款的来源平台和账号
- 该爆款的核心结构（钩子/观点/形式）
- 嫁接到未来流量的具体角度

不得只写"参考了某爆款"而不说明结构和嫁接方式。

**R8. 类型占比匹配阶段**
每周选题单必须包含三类用途：**吸引注意 / 展示专业 / 推动转化**。三类占比必须符合 STAGE_PLAYBOOK.md 中当前阶段的规定。

**R9. 字段完整**
每条选题的表格字段必须完整：优先级 / 目标平台 / 目标决策人 / 预期作用 / 来源 / 业务价值 / 钩子 / 核心角度。缺字段不合格。

---

## 三、内容质量红线（两个产出都适用）

**R10. 行业专属**
必须是入境游行业专属内容。通用营销内容、通用 AI 科普、通用出海内容，一律不合格。

**R11. 观点密度**
每条选题必须有"可被引用的观点或框架"，不能只是信息罗列或热点复述。

**R12. 无黑话**
不得出现"赋能""闭环""抓手""心智占位""高维度""数字化转型"等黑话。不得出现"在激烈市场竞争下""随着……的不断发展""综上所述"等 AI 味/报告体。（详见共享 knowledge 中的 BRAND-VOICE.md 与 FORBIDDEN.md）

**R13. 无过度承诺**
不得承诺具体排名、询盘数、ROI 倍数；不得使用"包成交""XX 天速成"等话术；涉及效果只能引用真实案例的具体数据。

**R14. 政策与数据合规**
涉及签证/免签/政策条款必须标注"建议核实"；涉及行业数据必须注明权威来源（携程/移民局/UNWTO 等）；不得以客户名义发布任何声明。

---

## 不合格则重做

Agent 04 在产出完成后，应自行对照 R1-R14 自检。任一条未满足，在输出前自行修正。不得输出不合格产出并以"仅供参考"免责。
