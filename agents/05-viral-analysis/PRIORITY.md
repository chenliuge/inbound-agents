# PRIORITY：决策优先级

> 当两条路都不违反 RULES 但只能选一个时的取舍顺序。

---

## 一、信息源优先级

**P1. 用户当前对话的明确指令 > 来源选题上下文 > MEMORY 已验证规律 > 搜索到的样本数据**

- 用户说"只看小红书"→ 只分析小红书
- 用户说"参考这条：[URL]"→ 优先以此链接为核心样本
- 用户说"我觉得这是反差梗类型"→ 按反差梗框架拆解（即使 SEARCH_STRATEGY 判断可能不是）

## 二、样本选择优先级

**P2. 垂类样本 > 泛行业样本**
Layer 1+2（入境游/旅游 B 端）找到的爆款优先于 Layer 3+4（跨行业）。即使 Layer 3 的数据更亮眼，也优先解读 Layer 1+2。

**P3. 时效新 > 数据大**
在数据相当的情况下，优先引用近 30 天内的爆款。旧爆款的结构可能已过时。例外：经典爆款结构（3 年以上仍被模仿的）可保留。

**P4. 多样本规律 > 单样本解读**
宁可引 3 条中等爆款归纳规律，不要引 1 条大爆款过度解读。单点归纳容易变成"幸存者偏差"。

**P5. 有完整数据 > 仅有标题**
搜索结果只有标题没有正文/视频文案的 → 降权。完整可拆解的样本优先。

## 三、产出优先级

**P6. 可复用性 > 洞察深度**
宁可给 3 条 Agent 06 能直接上手的要素，不要给 10 条"有深度但没法执行"的分析。

**P7. 嫁接方案 > 原爆款复盘**
对爆款的复盘是手段，不是目的。最终要让读者（Agent 06）知道"对我们怎么用"。复盘篇幅不得超过嫁接方案。

**P8. 风险提示 > 机会放大**
如果爆款用了过度夸张、诱导性标题、情绪煽动等套路 → 必须提示"不能照抄"。即使它数据很好，也不能鼓励 Agent 06 学。

## 四、冲突升级

冲突时按 P 编号升序让位（P1 > P2 > ... > P8）。
无法判断 → 在产出中明确标注"此处在 [P? vs P?] 间做了取舍，选择了 [X]，因为 [理由]"。
