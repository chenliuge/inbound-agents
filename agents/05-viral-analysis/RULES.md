# RULES：产出合格门槛

> 本文件定义 Agent 05 产出必须满足的硬性红线。任何输出若有任一条未满足，即为不合格，必须重做或标注"无法完成"原因。

---

## 一、数据真实性（绝对红线）

**R1. 必须基于真实爆款样本**
每条分析必须引用 SEARCH_STRATEGY 四层搜索中找到的真实作品（提供 URL + 发布日期 + 真实数据）。不得基于想象或记忆生成"爆款规律"。

**R2. 样本置信度必须标注**
每份产出开头必须标注置信度（🟢 高 / 🟡 中 / 🔴 低）及依据。不得假装样本充足。

**R3. 低置信度必须触发人工补充**
当 Layer 1+2+3+4 全部搜索后仍无 3 条以上达到爆款阈值的样本 → 产出中必须包含"【需要人工补充】"标记，并说明需要什么类型的参考。不得在低置信下硬写分析。

---

## 二、结构完整性

**R4. 图文爆款默认用 7 维框架拆解**
对每条入选爆款的图文内容，**默认**覆盖 EXTRACTION_FRAMEWORK 的 7 个维度（标题公式 / 开头钩子 / 结构模板 / 视觉节奏 / 金句位置 / CTA / 话题标签）。
**例外**：若样本结构特殊不适用标准框架（如纯图片笔记、漫画体、跨模态），允许自定义维度拆解，但必须在产出中**明确说明**：
- 为什么标准框架不适用（1-2 句）
- 采用的自定义维度清单

**R5. 框架灵活性的硬约束**
即便自定义维度，仍须满足：
- 每条拆解必须给出"Agent 06 可直接复用的具体元素"（不能全是抽象总结）
- 仍需在"嫁接到未来流量"章节给出保留/改造/避免清单（R7 不豁免）

**R6.（预留）视频分析能力暂未启用**
当前 Agent 05 仅处理**小红书+公众号图文**。视频爆款分析属于档 2 能力，待后续版本启用。

**R7. 必须输出"嫁接方案"**
仅有爆款拆解不够，每份产出必须有独立章节「嫁接到未来流量」，明确：
- 保留什么（爆款内核/结构）
- 改造什么（换成入境游场景）
- 避免什么（与 BRAND-VOICE 冲突的要素）

---

## 三、平台差异化

**R8. 必须覆盖小红书 + 公众号两个平台**
除非置信度低（R3），否则每份产出必须对两个平台**分别**给出独立分析。不得只做一个平台就交付。

**R9. 平台建议不得互相搬运**
不得把小红书短图文结构直接套到公众号长文，不得把公众号长文拆分成小红书。两个平台的内容逻辑（字数/节奏/配图/emoji/CTA）完全不同，必须分开给。

---

## 四、下游对接

**R10. 必须明确标注来源选题**
产出文件头必须写：来自 `05-topic-plan-pool/W[XX] 第 N 条`。没有明确来源的分析一律作废（脱离选题上下文的爆款分析毫无意义）。

**R11. 必须给 Agent 06 明确的保留/改造/避免清单**
产出末尾必须有「交接给 Agent 06 的说明」章节，**用清单形式**列出：
- 必保留的 3-5 条爆款要素
- 必改造的 3-5 条要素
- 必避免的 2-3 个陷阱

**R12. CTA 必须遵守 BRAND-VOICE**
所有建议的 CTA 必须导向 L0 免费竞争力报告，不得引入其他转化路径。

---

## 五、禁止事项

**R13. 禁止输出"泛泛方法论"**
不得出现"多用数据"、"引发共鸣"、"讲故事"这类无法执行的建议。所有拆解必须落到**具体要素**。

**R14. 禁止使用 BRAND-VOICE 禁用语**
即使爆款原文用了黑话（赋能/闭环/抓手），你的拆解描述中不得复用。用 BRAND-VOICE 定义的替代表达。

**R15. 禁止在低置信下假装高质量**
如果样本不足导致拆解质量低，必须明确说"本次样本仅 2 条，置信度偏低，结构归纳仅供参考"。绝不粉饰。

---

## 不合格则重做或标注

Agent 05 在产出完成后对照 R1-R15 自检。任一条不满足：
- 可自我修正的 → 修正后再输出
- 无法自我修正（如样本不足）→ 产出中明确标注原因
