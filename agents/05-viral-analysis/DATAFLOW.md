# DATAFLOW：数据流转

> 定义 Agent 05 的输入、输出、触发方式和协作关系。
>
> **当前版本范围**：仅处理 **小红书 + 公众号图文爆款**。视频爆款能力暂未启用。

---

## 一、触发方式（三种模式）

Agent 05 由用户对话指令触发，支持三种模式：

### 模式 B（默认推荐）：批量分析本周选题

从 05-topic-plan-pool/approved/ 读取最新一期周选题，对每条标记「需爆款分析 ✅」的选题做分析。

```
@爆款分析官                      # 什么都不加 → 批量
@爆款分析官 批量分析本周选题
```

**Agent 05 会**：
1. 找 05-topic-plan-pool/approved/ 里修改时间最新的文件
2. 解析其中的 markdown 选题表（列头含"选题标题"的那张）
3. 筛选出「需爆款分析 ✅」的行
4. 对每条执行完整流程（四层搜索 + LLM 拆解）
5. 合并输出为一个 md 文件写入 06-viral-analysis-pool/pending/

**兼容性**：若选题表无「需爆款分析」列（老数据），按「来源类型 ∈ {爆款拆解, 外部热点, 需求池}」判定。

### 模式 B-N（指定第 N 条）

```
@爆款分析官 第 3 条
```

无视「需爆款分析」列，强制分析本期第 3 条选题。

### 模式 A（手动）：单选题临时分析

用户直接给一个选题字符串，Agent 05 只搜这个字符串、不读选题池。

```
@爆款分析官 入境游 GEO 4 步法
```

**用于**：临时探索、复盘某个选题、demo 测试。

### 模式 C（待实现）：粘贴 URL 拆解

```
@爆款分析官 拆这条：https://www.xiaohongshu.com/explore/abc123
```

## 二、输入源

### 2.1 启动时必读（常驻上下文）

| 文件 | 用途 |
|---|---|
| IDENTITY.md | 角色定义 |
| RULES.md | 产出合格门槛 |
| PRIORITY.md | 决策优先级 |
| SEARCH_STRATEGY.md | 四层搜索策略 |
| VIRAL_THRESHOLDS.md | 爆款阈值 |
| EXTRACTION_FRAMEWORK.md | 拆解框架 |
| MEMORY.md | 历史记忆 |
| config/wechat_accounts.yaml | 参考账号池 |

### 2.2 公司共享知识

| 文件 | 用途 |
|---|---|
| shared/knowledge/COMPANY.md | 公司业务、产品 |
| shared/knowledge/USER.md | 目标客户画像 |
| shared/knowledge/BRAND-VOICE.md | 语气规范（拆解后的嫁接要符合这个） |
| shared/knowledge/FORBIDDEN.md | 全局禁区（不得嫁接涉及这里的要素） |

### 2.3 选题上下文（模式 B 必读）

| 来源 | 取用规则 |
|---|---|
| 05-topic-plan-pool/approved/（最新一期） | 本周完整选题表 |
| 04-strategy-pool/approved/（当季） | 当季策略框架（辅读） |

### 2.4 平台数据（通过 scripts/social_crawler.py）

四层搜索按需调用：
- 小红书：`xhs_search_note` + `xhs_note_detail`（拿完整正文）+ `xhs_user_post`（按账号池）
- 公众号：`wechat_kw_search` + `wechat_hot_typical_search` + `wechat_post_history`（按账号池）+ `wechat_read_zan`（补阅读数据）

### 2.5 视频下载（档 2，暂未启用）

视频能力待后续版本启用。当前不处理抖音/视频号样本。

## 三、输出路径

### 3.1 输出池

所有产出写入 `collaboration/06-viral-analysis-pool/pending/`。

### 3.2 文件命名

- 模式 A：`viral-[选题短名]-YYYY-MM-DD_HHMM.md`
- 模式 B：`viral-batch-W[XX]-YYYY-MM-DD_HHMM.md`（多选题合并一个文件）
- 模式 C：`viral-manual-YYYY-MM-DD_HHMM.md`

### 3.3 与 Agent 06 的衔接

Agent 06（内容制作官）每次写内容前，读取 `06-viral-analysis-pool/approved/` 下对应选题的拆解文件作为创作依据。

## 五、与其他 Agent 的协作图

```
Agent 04 (周选题产出)
   ↓  包含"需爆款分析"标记的选题
[人工 / 自动] 触发 Agent 05
   ↓
Agent 05:
   1. 读选题 / 读用户指令
   2. 四层搜索（scripts/social_crawler.py）
   3. 档 1 拆解（文字元数据）
   4. 档 2 拆解（视频：yt-dlp 下载 + Whisper 转写）
   5. 输出到 06-viral-analysis-pool/pending/
   ↓  人工审核通过 → approved/
Agent 06 (内容制作官)
   读取 06-viral-analysis-pool/approved/ 作为创作依据
```
