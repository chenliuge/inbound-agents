# Agent 01 · 数据流转

## 输入来源

| 来源 | 类型 | 说明 |
|------|------|------|
| DuckDuckGo | 搜索引擎 | 免费，覆盖英文/中文网页 |
| Tavily | 搜索引擎 | 需 API Key，结果质量更高 |
| 抖音（cn8n API） | 社交媒体 | 关键词视频搜索，含播放/点赞数据 |
| 小红书（cn8n API） | 社交媒体 | 笔记搜索，含互动数据 |
| 微信公众号（cn8n API） | 行业媒体 | 指定账号历史文章抓取 |

## 处理流水线

```
queries.yaml
    ↓ [Planner] 按频率过滤查询词
    ↓ [Searcher] 5源并发搜索（asyncio + ThreadPoolExecutor）
    ↓ [Compressor] URL去重 → 时效过滤（14天）→ 来源分级打标
    ↓ [Summarizer] LLM生成三维结构化简报
    ↓ [Reporter] 拼接元数据头部 + 数据透明度尾部
    → collaboration/01-intel-pool/pending/
```

## 信号评分规则（LLM内部执行）

| 维度 | 权重 | 说明 |
|------|------|------|
| 重要性 | 40% | 直接影响业务；tier_1/tier_2 来源加权 |
| 新颖性 | 25% | 首次出现 > 已知趋势延续 |
| 可行动性 | 20% | 能在本周/本月转化为具体行动 |
| 证据强度 | 15% | 多源交叉验证 > 单一来源 |

## 引用规则

| 来源层级 | 引用方式 |
|---------|---------|
| tier_1（官方） | 直接陈述为事实，"XXX官方公告显示" |
| tier_2（专业媒体） | 直接陈述为事实，"据环球旅讯报道" |
| tier_3（自媒体） | 必须标注"据XX报道"或"XX称" |
| tier_4（社交） | 必须标注"用户反馈显示"或"社交数据显示" |
